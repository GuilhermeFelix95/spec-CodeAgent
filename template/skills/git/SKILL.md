# Skill: Git

Use when the task involves repository history or local version control operations.

## When to use

- Creating or switching branches.
- Inspecting diffs, history, or staged changes.
- Preparing a clean commit.
- Resolving merge or rebase conflicts.

## Checklist

- Keep commits focused and intentional.
- Review the diff before committing.
- Preserve unrelated user changes.
- Use safe, non-destructive operations.
- Prefer the smallest viable history change.

## Guardrails

- Do not rewrite history casually.
- Do not discard unowned changes.
- Do not mix unrelated work in one commit.
