# Skill: Pull Request

Use when a change needs to be packaged for review and merge.

## When to use

- Preparing a branch for review.
- Summarizing change scope and risk.
- Reviewing PR readiness and merge criteria.
- Tracking follow-up work after review.

## Checklist

- State what changed and why.
- Link the spec, tickets, or issue if they exist.
- Verify tests, checks, and review expectations.
- Keep the PR small and coherent.
- Call out any risk, migration, or rollout concern.

## Guardrails

- Do not hide unresolved work in the PR.
- Do not mix unrelated refactors into feature work.
- Do not skip validation before requesting review.
