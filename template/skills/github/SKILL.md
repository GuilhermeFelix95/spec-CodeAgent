# Skill: GitHub

Use when the task involves GitHub as a hosting or collaboration layer.

## When to use

- Reading or summarizing a repository or PR.
- Opening, updating, or reviewing issues and pull requests.
- Inspecting checks, reviews, or merge status.
- Linking code changes to hosted collaboration artifacts.

## Checklist

- Confirm the target repo, branch, and PR or issue.
- Prefer direct source data over assumptions.
- Keep comments concise and actionable.
- Match the platform's collaboration flow.
- Verify permissions before writing externally.

## Guardrails

- Do not post outside the approved scope.
- Do not assume a PR is ready without checks.
- Do not mix review feedback with implementation tasks.
