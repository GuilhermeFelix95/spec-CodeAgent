# Skill: DevEx Bootstrap

Use when the goal is to make the project easier to start, understand, and operate.

## When to use

- New repository setup.
- Onboarding flow or local environment friction.
- Repeated manual steps that should be scripted.
- Standardizing how people run the project.

## Checklist

- Standardize the basic commands.
- Keep local setup simple.
- Explain the development flow clearly.
- Reduce repetitive manual steps.
- Preserve compatibility with the existing stack.

## Guardrails

- Do not overbuild the setup.
- Do not require hidden knowledge to start.
- Do not break existing contributor workflows.
